# fix-032c0404d14f8baa — updateBooking / updateBookingStatus create phantom partial records for non-existent booking IDs

**Weight:** 0.0000 (share of project budget)
**Reward:** 0 TBLBOT

Neither `updateBooking` (line 255–270) nor `updateBookingStatus` (line 272–277) checks whether the booking exists before calling `this.redis.hset(KEY.booking(id), …)`. If the ID references a booking that was already deleted (or never created), Rediset `HSET` will create a new hash containing ONLY the update fields — no `id`, `ref_code`, `date`, or indexes. Subsequent `getBooking` reads this corrupted record and returns a booking with default-zero/empty values for all missing fields. `deleteBooking` (line 280) correctly gates on existence with `if (!b) return`, but the update path has no equivalent guard, opening a data-corruption window from race conditions or calling mistakes.
