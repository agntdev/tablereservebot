# fix-8001626af84e3c0f — updateBooking does not re-index when date changes, corrupting bookings-by-date secondary index

**Weight:** 0.0000 (share of project budget)
**Reward:** 0 TBLBOT

`updateBooking` (src/storage/index.ts:255–270) allows `updates.date` but only writes it to the booking's Redis hash — it never removes the booking from `bookings:by-date:{old_date}` nor adds it to `bookings:by-date:{new_date}`. After a date change, `listBookingsByDate(new_date)` will not include the booking and `listBookingsByDate(old_date)` will still return it (stale). The sibling method `deleteBooking` (line 279–290) reads the old date and cleans up the index, establishing the expected pattern. A caller changing `date` via `updateBooking` will silently corrupt the secondary index.
