# fix-8e89641ee400eb70 — Stale button text in E4T1, E4T2, E4T3 test specs — mismatch with mainMenu()

**Weight:** 0.0000 (share of project budget)
**Reward:** 0 TBLBOT

`mainMenu()` in `src/bot.ts:47-52` produces buttons with text `"📋 How to Reserve"` (callback `menu:about`) and `"🛟 Need Assistance?"` (callback `menu:help`). However, the test specs for **E4T1** (`tests/specs/E4T1.json`), **E4T2** (`tests/specs/E4T2.json`), and **E4T3** (`tests/specs/E4T3.json`) still reference the **old** button text `"📋 About"` and `"🛟 Help"`. These will fail at test time because the expected inline keyboard payloads do not match the actual output of the bot.

By contrast, E4T4 and E4T5 test specs use the **correct** button text (`"📋 How to Reserve"`, `"🛟 Need Assistance?"`), confirming that `mainMenu()` was updated but the E4T1/E4T2/E4T3 specs were not.

**Fix:** Update the three affected test specs to use `"📋 How to Reserve"` and `"🛟 Need Assistance?"` in their expected `inline_keyboard` payloads.
