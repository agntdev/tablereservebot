# fix-866b64d8bb03c3f1 — Missing guest name in /book confirmation when name is provided in command

**Weight:** 0.0000 (share of project budget)
**Reward:** 0 TBLBOT

When using `/book <date> <time> <size> <guest_name>`, the handler at `src/bot.ts:670-678` creates the booking with the guest name stored correctly, but the confirmation message omits the `Name:` (and `Phone:`) line entirely. The `guestName` variable is available (line 605) but never interpolated. This is inconsistent with the interactive calendar flow (`finalizeBooking` at `src/bot.ts:190-200`) which correctly includes both `Name:` and `Phone:` in the confirmation reply.
