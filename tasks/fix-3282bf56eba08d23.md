# fix-3282bf56eba08d23 — Midnight-crossing bookings cause overlap detection failure (double-booking bug)

**Weight:** 0.0000 (share of project budget)
**Reward:** 0 TBLBOT

`allocateFirstFit` in `src/allocate.ts` uses `timeToMinutes()` which converts `HH:MM` to flat minutes without day-wrapping awareness. When `generateSlots()` in `src/slots.ts` produces slots that span midnight (e.g. `close_time` ≤ `open_time`, yielding slots like `23:00–01:00`), the stored booking `end_time` wraps around (e.g. `"01:00"`). `timesOverlap("23:00","01:00", "00:00","01:00")` computes `1380 < 60 && 60 > 0` = `false`, missing the actual overlap. This allows two bookings with midnight-crossing intervals to be created for the same tables without any overlap detection — a direct double-booking vulnerability.

**Location:** `src/allocate.ts:8-10` (`timeToMinutes`), `src/allocate.ts:13-24` (`timesOverlap`), `src/slots.ts:30-34` (`formatTime` wraps into 00:00–23:59 range).
