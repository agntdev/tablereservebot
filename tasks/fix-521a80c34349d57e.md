# fix-521a80c34349d57e — E5T1: Reminder time calculation ignores venue timezone

**Weight:** 0.0000 (share of project budget)
**Reward:** 0 TBLBOT

`checkAndSendReminders` in `src/reminder.ts:43-50` uses `Date.UTC()` to compute the booking timestamp from the date string and start time. It then compares against `now.getTime()` (also in UTC). The venue's configured timezone (`settings.timezone`) is never used. For venues not in UTC, the `timeUntilMs` will be off by the UTC offset, causing reminders to fire at the wrong wall-clock time (e.g., 2 hours early for UTC+2 venues, or 2 hours late for UTC-2).
