# fix-9e839d40a9ff6038 — Calendar slot picker is a dead-end — no path to booking after slot selection

**Weight:** 0.0000 (share of project budget)
**Reward:** 0 TBLBOT

The `slot:pick:` callback at `src/bot.ts:323-337` confirms the slot selection with a message like `✅ Date: 2026-06-15\nGuests: 4\nTime: 19:00` but offers **no button or mechanism to proceed to booking**. The session field `selectedSlot` is stored but never consumed by any booking handler.

The E1T4 slot-discovery flow (calendar → party size → slots → pick) and the E1T5 booking flow (`/book`) operate as entirely disconnected paths. A user who navigates the full calendar UI to find an available slot must discard that result and manually construct a `/book` command.

**Fix:** After slot selection, either offer an inline "Book this slot" button that transitions into the name/phone collection flow, or wire `selectedSlot`/`selectedDate`/`partySize` into the booking handler so the calendar flow can feed directly into `finalizeBooking`.
