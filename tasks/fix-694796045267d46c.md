# fix-694796045267d46c — Non-atomic reschedule: tables freed before new booking committed

**Weight:** 0.0000 (share of project budget)
**Reward:** 0 TBLBOT

In `finalizeReschedule` at `src/bot.ts:266`, the original booking's tables are freed (`allocated_tables: []`, status changed to `"rescheduled"`) **before** the new booking is created at `src/bot.ts:285-291`. If the Node process crashes or is killed between these two operations:

1. The original booking has its tables released.
2. The new booking was never created.
3. The rollback at line 293 (`updateBooking(original.id, ...)`) only executes on an **allocation failure**, not on a crash.

Result: table allocations are permanently lost for that booking.
